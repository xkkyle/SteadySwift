## Button

### 3가지 표현 방법

```swift
Button("기본 버튼") {
	print("press 기본 버튼")
}

Button(action: { print("press 버튼 2") }) {
	Text("버튼 2")
		.padding()
		.background(RoundedRectangle(cornerRadius: 10).strokeBorder())
}

Button(action: { print("press 버튼 3") }) {
	Circle()
		.stroke(lineWidth: 2)
		.frame(width: 80, height: 80)

		)
```

## Navigation View

- 네비게이션 뷰 : 네비게이션 스택을 이용해 콘텐츠 뷰들을 관리하는 컨테이너 뷰
  - UINavigationController 또는 UISplitViewController 역할을 수행
  - 기능
    - 스택 구조의 내비게이션 계층을 구성하고 화면을 전환
    - 네비게이션 바 아이템, 타이틀 같은 요소들을 활용하여 UI를 구성하는데 활용할 수 있음

```swift

// 잘못된 방법
Navigationview { ... }.navigationBarTitle("내비게이션 바 타이틀")

// 옳은 방법

NavigationView {
	Image()
		.navigationBarTitle("내비게이션 바 타이틀", displayMode: .large) // automatic(기본값), .large, .inline
}
```

- 네비게이션 뷰에 직접 설정하는 형태라면
  - 네비게이션 뷰 스택에 포함된 모든 자식 뷰가 동일한 제목만을 사용하게 된다.
  - UIKit에서도 내비게이션 뷰컨트롤러 자체에 값을 넣는 것이 아니라, 뷰 컨트롤러에 제목을 직접 설정해주었던 것
    - e.g. `view.title = "제목입니다"`
- 네비게이션 뷰에 사용되는 수식어들은 Preference라는 기능이 사용되어 하위 뷰가 상위 뷰에 데이터를 전달하는 방식으로 적용된다.

> [!info] 구조체(View 프로토콜 상속받은) `var body: some View { ... }` 는 계산 속성
> `var body: some View { ... }` 내부에 있는 `let leadingItem = Button(action : ) { ... }`는 단순 상수이다.

### GeometryReader

- 지오메트리 리더는 자식 뷰에 부모 뷰와 기기에 대한 크기 및 좌표계 정보를 전달하는 기능을 수행하는 컨테이너 뷰
  - 폰이 회전하는 경우처럼 뷰의 크기가 변해도 그 값이 자동으로 갱신된다.
  - `init(@ViewBuilder content: @escaping (GeometryProxy) -> Content)`
    - content에는 스택과 같이 ViewBuilder 속성이 선언되어 있어, 뷰를 나열하는 것만으로도 사용할 수 있다.
      - 이 때, 뷰가 나열되는 방식은 ZStack처럼 쌓아올리는 효과
- GeometryReader 내에 자식 뷰를 하나만 배치하면, Circle, Rectangle처럼 화면을 자동으로 꽉채우는 효과가 있다.

```swift
        GeometryReader { _ in
            Circle()
                .fill(.blue)
                .frame(width: 350, height: 350)

            Circle()
                .fill(.orange)
                .frame(width: 280, height: 280)

            Circle()
                .fill(.purple)
                .frame(width: 200, height: 200)
            Text("Top Leading")
                .font(.largeTitle)
        }.background(.gray)
```

### GeometryProxy

#### GeometryProxy는 2개의 property, 1개의 method, 하나의 첨자(subscript)를 제공하며, GeometryReader의 정보를 자식 뷰에게 전달가능하다.

```swift
struct GeometryProxy {
	var size: CGRect { get }
	var safeAreaInsets: EdgeInsets { get }
	func frame(in coordinateSpace: CoordinateSpace) -> CGRect
	subscript<T>(anchor: Anchor<T>) -> T { get }
}
```

```swift
 GeometryReader { geometry in
            Text("Geometry Reader")
                .font(.largeTitle)
                .bold()
                .position(x : geometry.size.width / 2, y: geometry.safeAreaInsets.top)

            // .position -> GeometryProxy
            // size
            // safeAreaInsets
            // frame(in:) -> 단순 CGRect 정보가 아닌 CoordinateSpace라는 열거형 타입이 가진 3가지 값중 하나를 지정하면, 그 좌표 공간에 대한 정보를 반환
            // subscript (첨자)
            VStack {
                Text("Size").bold()
                Text("Width: \(Int(geometry.size.width))")
                Text("Height: \(Int(geometry.size.height))")
            }.position(x: geometry.size.width / 2 , y: geometry.size.height / 2.5)

            VStack {
                Text("SafeAreaInsets").bold()
                Text("Top: \(Int(geometry.safeAreaInsets.top))")
                Text("Bottom: \(Int(geometry.safeAreaInsets.bottom))")
            }.position(x: geometry.size.width / 2 , y: geometry.size.height / 1.4)
        }
        .font(.title)
        .frame(height: 500)
        .border(Color.green, width: 5)
```

#### `GeometryProxy`는 `frame` 에 대한 정보도 전달한다.

여기서의 frame은 자신의 `CGRect` 정보를 전달하는 것이 아닌 `CoordinateSpace` 열거형 정보를 전달한다.

- global : 화면 전체 영역 (윈도우 bounds) 를 기준으로 한 좌표 정보
- local : 현재 지오메트리 리더의 bounds를 기준으로 한 좌표 정보
- named("VStackCS") : 명시적으로 이름을 할당한 공간을 기준으로 한 좌표 정보
  - `coordinateSpace(name : "VStackCS")`

```swift
enum CoordinateSpace {
	case local
	case global
	case named(AnyHashable)
}
```

```swift
//
//  GeometryProxyPractice.swift
//  HelloSwiftUI
//
//  Created by KWON HYUKMIN on 6/24/26.
//

import SwiftUI

struct GeometryProxyPractice: View {

    func stringFormat(for point: CGPoint) -> String {
        return String(format : "(x: %.f, y: %.f)", arguments: [point.x, point.y])
    }

    // global : 화면 전체 영역 (윈도우 bounds)를 기준으로 한 좌표 정보
    // local : 지오메트리 리더의  bounds를 기준으로 한 좌표 정보
    // named("VStackCS") : 명시적으로 이름을 할당한 공간을 기준으로 한 좌표 정보
    func contents(geometry g: GeometryProxy) -> some View{
        VStack {
            Text("Local").bold()
            Text(stringFormat(for : g.frame(in: .local).origin)).padding(.bottom)

            Text("Global").bold()
            Text(stringFormat(for : g.frame(in: .global).origin)).padding(.bottom)


            Text("Named VStackCS").bold()
            Text(stringFormat(for : g.frame(in: .named("VStackCS")).origin)).padding(.bottom)

            Text("Named HStackCS").bold()
            Text(stringFormat(for : g.frame(in: .named("HStackCS")).origin)).padding(.bottom)
        }
    }

    var body: some View {

        HStack {
            Rectangle().fill(.yellow).frame(width: 30)

            VStack {
                Rectangle().fill(.blue).frame(height: 200)

                GeometryReader {
                    // 레이아웃을 위한 뷰와 실제 컨텐츠 분리
                    self.contents(geometry: $0)
                }
                .background(.green)
                .border(.red, width: 4)


            }.coordinateSpace(name : "VStackCS") // VStack 좌표 공간에 이름 부여
        }.coordinateSpace(name : "HStackCS") // HStack 좌표공간에 이름 부여
    }
}

#Preview {
    GeometryProxyPractice()
}

```

### Frame 수식어

- UIKit 에서는 뷰에 프레임을 설정한다. (제약조건[constraint]을 설정)
- SwiftUI에서는 뷰를 직접 변경하는 것이 아니라, 원래의 뷰를 수식하는 새로운 뷰를 반환한다.
  - 액자처럼 콘텐츠를 담고 있는 하나의 View

```swift
Text("Swift") // Text 타입
Text("Swift").frame(width : 200) // ModifiedContent<Text, _FrameLayout>
```

#### `frame`은

- 자식뷰가 **사용가능한 크기를 제안**하기 위해 사용
- 뷰의 **정렬 위치**를 결정하는 역할
- 다만, 제한된 공간내에서 실제로 자식 뷰가 어느 정도 크기를 갖고 보이게 될지는, 그 뷰가 직접 결정한다.

```swift

Text("Yellow Space").background(.yellow).frame(width: 120, height: 80) // 문자 사이즈의 크기만큼만 배경을 칠하며 차지한다.

Rectangle().fill(.green).frame(width: 200, height: 100) // 공간 전체를 활용

```

> 위와 같이 뷰가 가진 성질에 의해 결정된다.

#### `frame(width:height:alignment)`

- alignment 는 `.leading`, `.trailing`, 등의 열겨헝 타입을 사용할 수 있고, 어떤 인수도 전달하지 않는 경우 중앙(`.center`)에 배치된다.

```swift
HStack {
```

#### SwiftUI에서 뷰 레이아웃 과정

1. 부모 View가 활용가능한 크기를 자식 View에게 제안한다.
2. 자식 뷰는 그 자신의 크기를 결정한다.
3. 부모 뷰는 자신의 좌표 공간에서 자식의 뷰를 적절하게 배치한다.

#### 고정 크기 vs 크기 제약 조건

width, height로 고정된 크기를 뷰에 부여할 수 있었다.

##### 크기 제약 조건

- 고정된 값을 입력하는 대신 최소(min)나 최대(max) 또는 이상적인 (ideal) 값에 대한 제약조건을 입력하는 방법도 있다.

```swift
func frame(minWidth: CGFloat? = nil, idealWidth: CGFloat? = nil, maxWidth: CGFloat? = nil, minHeight: CGFloat? = nil, idealHeight: CGFloat? = nil, maxWidth: CGFloat? = nil, alignment: Alignment = .center ) -> some View
```

- 주의할 점
  - min < ideal < max, 즉 크기 순서대로 (오름차순) 값을 지정해야 한다.

```swift

HStack {
	// width: minWidth < maxWidth
	Rectangle().fill(.red).frame(minWidth: 100)
	Rectangle().fill(.yellow).frame(maxWidth: 15)

	// height : minHeight < maxHeight
	Rectangle().fill(.yellow).frame(minHeight: 150)

	Rectangle().fill(.yellow).frame(maxHeight: .infinity)


	Rectangle().fill(.blue)
		.frame(maxWidth: .infinity, maxHeight: .infinity    )

	Rectangle().fill(.purple)
}.frame(width: 300, height: 150)

```

#### 이상적인 크기 (Ideal Size) 와 FixedSize 수식어

- 이상적인 크기는 UIKit에서의 고유 컨텐츠 크기 (Intrinsic Content Size) 같은 개념으로 이해하면 편하다. (✅ idealWidth, idealHeight 값을 넣어봤다면 아무런 변화가 없는 이유)
- 각각의 뷰는 그 특성에 맞는 고유한 컨텐츠 크기를 갖고 있어, AutoLayout에 활용할 수 있었다.
- fixedSize()라는 수식어를 활용하면 이 크기 정보(idealWidth, idealHeight)를 활용할 수 있다.

```swift
VStack {

            Text("Frame Ideal")
                .font(.title)
                .bold()
                .frame(width: 80, height: 30)

            Rectangle()
            Color.red
            Image(systemName: "book.fill")

}

VStack {

	Text("Frame Ideal")
		.font(.title)
		.bold()
		.fixedSize()
		.frame(width: 80, height: 30)

	Rectangle().fixedSize()
	Color.red.fixedSize()
	Image(systemName: "book.fill").resizable().fixedSize()

}

// frame 수식어로 idealWidth를 설정하고, fixedSize 수식어를 사용하면 원하는 크기로 설정할 수 있다.
VStack {
	Text("Frame Ideal")
		.font(.title)
		.bold()
		.frame(idealWidth: 120)
		.fixedSize()

}

// fixedSize(horizontal:vertical)
// 앞서 본 .fixedSize()는 horizontal과 vertical 모두에 true를 준 것과 같다.

VStack {
	Group { // 자식 뷰에 공통으로 동일한 수식어를 적용하고 싶을 때 Group 활용 가능
		Text("FixedSize를 사용하면 글자가 생략되지 않습니다")
		Text("FixedSize를 사용하면 글자가 생략되지 않습니다").font(.title)
			.fixedSize(horizontal: true, vertical: false)
		Text("FixedSize를 사용하면 글자가 생략되지 않습니다").font(.title)
			.fixedSize(horizontal: false, vertical: true)
	}
	.font(.title)
	.frame(width: 150, height: 40)
}

```

#### LayoutPriority

`func layoutPriority(_ value: Double) -> some View`

- 레이아웃의 우선순위가 높을 경우 부모 레이아웃(parent layout)이 자식에 공간을 할당하는 데 있어서 형제 뷰 그룹 내에서 우선권을 갖는다.
- 부모 공간이 늘어날 때 더 빨리 늘어나고, 줄어들 때는 더 늦게 줄어든다.
- 모든 뷰는 기본적으로 우선권이 0으로 설정되며, 아주 조금이라도 차이가 난다면 값의 크기와 관계없이 우선순위가 달라진다.

```swift
// 1. no layout priority
HStack {
	Color.red
	Color.yellow
	Color.blue
}.frame(height: 40)


// 2. has Priority
HStack {
	Color.red.layoutPriority(1)
	Color.green
	Color.blue.layoutPriority(1)
}.frame(height: 40)

// 3. with min, max
HStack {
	Color.red.layoutPriority(1)
	Color.green.frame(minWidth: 30)
	Color.blue.frame(maxWidth: 50).layoutPriority(1)
}.frame(height: 40)

// 4. 고정크기 frame
HStack {
	Color.red.frame(width: 50)
	Color.green.layoutPriority(1)
	Color.blue.frame(maxWidth :50 ).layoutPriority(1)
}.frame(height: 40)
```

1. 우선순위를 결정하지 않는 경우 모두 동일한 크기로 할당된다.
2. red, blue 뷰에서만 우선순위를 1로 결정하면, green 뷰는 낮은 우선순위를 지닌다. 따라서, 크기를 지정하지 않는 한 공간을 전혀 할당받지 못한다. green 뷰가 보이지 않음에도 중간에 비어있는 공간이 생기는 것은 HStack의 Spacing 만큼 red-green, green-blue 사이의 spacing 공간이 2개 존재하기 때문이다. spacing : 0 으로 설정하면 green 공간이 없어지는 것을 확인할 수 있다.
3. minWidth를 갖는 green의 최소 너비를 확보한 것을 확인할 수 있다. blue 는 red와 동일한 우선순위이지만 maxWidth 를 설정하면 해당 값 이상의 크기를 차지하지 않아, red 뷰의 너비만 커진 모습이다.
4. 고정 크기 frame을 설정해주면 우선순위를 갖지 않고, 해당 크기를 가진다.
